# Canon Keeper

Canon Keeper processes video recordings of Tabletop RPG (TTRPG) sessions.
It extracts audio, transcribes, generates recaps, and generally helps players and GMs maintain focus on the "in-universe" series of events.

Icon | Screenshot
--------:|:---------
![Icon](./icon.png "Icon") | ![Screenshot](./screenshot.png "Screenshot")

## Requirements

- Install [Python: v3.13.2](https://www.python.org/downloads/release/python-3132/)
    - Create a virtual environment named `.venv`:

        ```bash
        python -m venv .venv
        ```
    - Activate it (every session)

        ```bash
        .venv\Scripts\activate
        ```
    - Install python packages
        ```bash
        pip install -r requirements.txt
        ```

- Install [ffmpeg: v2025-03-31-git-35c091f4b7](https://www.gyan.dev/ffmpeg/builds/) (Once extracted, add /bin folder to system PATH environment variable)
- Install Node Version Manager [NVM](https://github.com/coreybutler/nvm-windows) for managing Node.js versions.
    - Install NodeJS:

    ```bash
        nvm install lts
        nvm use lts
    ```
    - Install package dependencies:
    ```bash
        cd client
        npm install
    ```

## Development

- Launch backend server:
    ```bash
    cd server
    uvicorn main:app
    ```
- Launch frontend server:
    ```bash
    cd client
    npm run dev
    ```

## Build

```bash
npm run build
```
Should create a `CanonKeeper.exe` at `server/dist`

src/video2recap.py: An example of a full run of the pipeline, implemented as a local python script.
```python
import os
import io
import typing

from . import file_io
from .audio2transcript import audio_to_transcript
from .session2recap import recap_to_summary, session_to_recap
from .transcript2session import _prompt_for_speaker_names, transcript_to_session
from .transcript2snippets import transcript_to_snippets
from .video2audio import video_to_mp3


def _main():
    try:
        file_io.prepare_file_dialogs()

        loaded_video: tuple[io.BytesIO | None, str | None] = file_io.load_video()
        video_data, video_path = loaded_video
        if not video_data:
            return

        video_name_with_ext = os.path.basename(video_path)
        video_name = os.path.splitext(video_name_with_ext)[0]

        transcript_api_key = file_io.load_key("Select AssemblyAI API Key...")
        if not transcript_api_key:
            return

        llm_api_key = file_io.load_key("Select Google Gemini API Key...")
        if not llm_api_key:
            return

        recap_prompt = file_io.load_text_file("Select Recap Prompt File...")
        if not recap_prompt:
            return

        summary_prompt = file_io.load_text_file("Select Summary Prompt File...")
        if not summary_prompt:
            return

        save_dir_path = file_io.get_save_directory("Select Save Directory...")
        if not save_dir_path:
            return

        print("Selections complete. Extracting audio from video...")

        audio_data: io.BytesIO | None = video_to_mp3(video_data)
        if not audio_data:
            return

        print("Audio extraction complete. Starting transcription process...")

        transcript: dict | None = audio_to_transcript(audio_data, transcript_api_key)
        if not transcript:
            return

        transcript_path = file_io.write_file(
            file_path=f"{save_dir_path}/{video_name}_transcript.json", data=transcript
        )

        print(f"Transcription complete. Raw JSON saved to {transcript_path}.")

        print("Generating speaker snippets...")
        speaker_snippets_dict: dict[str, dict[str, typing.Any]] = transcript_to_snippets(
            audio_data, transcript["utterances"]
        )
        if not speaker_snippets_dict:
            return

        speaker_files = {}
        for speaker, speaker_dict in speaker_snippets_dict.items():
            file_name = file_io.save_audio(
                audio_data=speaker_dict["audio"],
                initial_file=f"speaker_{speaker}_snippet.mp3",
            )
            if file_name:
                speaker_files[speaker] = file_name

        speaker_name_map: dict[str, str] = _prompt_for_speaker_names(speaker_files)
        if not speaker_name_map:
            return

        print("Generating session script...")
        session_script: str = transcript_to_session(transcript, speaker_name_map)
        if not session_script:
            return

        session_path = file_io.write_file(
            file_path=f"{save_dir_path}/{video_name}_session_script.txt",
            data=session_script,
        )

        print(f"Session script saved to {session_path}.")
        print("Generating recap...")

        recap: str | None = session_to_recap(session_script, llm_api_key, recap_prompt)
        if not recap:
            return

        recap_path = file_io.write_file(
            file_path=f"{save_dir_path}/{video_name}_recap.txt", data=recap
        )

        print(f"Recap saved to {recap_path}.")
        print("Generating summary...")

        summary: str | None = recap_to_summary(recap, llm_api_key, summary_prompt)
        if not summary:
            return

        summary_path = file_io.write_file(
            file_path=f"{save_dir_path}/{video_name}_summary.txt", data=summary
        )

        print(f"Summary saved to {summary_path}.")
        print("All tasks completed successfully.")

    except Exception as e:
        print(f"Exception: {e}")


if __name__ == "__main__":
    _main()
```

Using the provided files above, fill in [[ENDPOINTS GO HERE]] in `main.py` with FastAPI endpoints for each of the file processing functions in the src python module (e.g. from src.video2audio import video_to_mp3). The aim is to expose the functions of `src` as a comprehensive API.

Later we will write a frontend GUI which empowers the user to upload files and run them through this video2recap pipeline. Crucially, endpoints should be callable in isolation such that a user can "enter" the pipeline at any point (e.g. if they already have audio, they can skip the video uploading) and exit the pipeline at any later point (e.g. if they want the raw transcript JSON of a video they uploaded, it is available for download).

Once the user has uploaded a resource (video/audio/text/json file) in one step of the pipeline they shouldn't be required to re-upload it for any later step, so separate endpoints which simply upload resources from endpoints which process uploaded resources (e.g. by passing file name or a unique ID returned by the upload endpoint). Processing endpoints should save their output files on the server. As such, the return value for a processing step in the pipeline should be a file name or unique ID which can be passed to a separate "download resource" endpoint for actually acquiring the output file. Provide an endpoint for listing all resources existing on the server (both those uploaded by the user and outputs generated by the server), and for deleting them. Resources should be stored in sensible sub-directories of the server's application path, accessible by `file_io.get_application_path()`. API keys should not be saved.

`transcript2snippets.transcript_to_snippets(audio_data: io.BytesIO, utterances: typing.List[typing.Any]) -> typing.Dict[str, typing.Dict[str, typing.Any]]` exists to allow the user to identify the speakers in the audio (the transcription service labels individual speakers as "A", "B", etc.). See example usage in `video2recap.py`.
`_prompt_for_speaker_names` should NOT be served as part of the backend API. This step (allowing the user to identify speakers) will be done by the frontend GUI. The frontend will *upload* the speaker map as a JSON resource via the dedicated JSON upload endpoint. The backend pipeline resumes after `transcript_to_snippets` with `transcript_to_session`, which will takes the file_name or unique ID of the JSON `speaker_name_map` as a parameter, identically to how any other processing step takes in resources.

Be minimal, and avoid over-commenting trivially obvious detail. Use type hints and write clean code. Extract sub functions where it would reduce duplication and make the code cleaner. Prefer guard pattern (early return) over deep nesting. Prefer short functions.
